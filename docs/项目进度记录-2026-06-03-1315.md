## 已经完成的功能

### 1. 说话动作素材库

现在已经把“说话时播放的动作片段”做成了素材库。素材放在：

```text
data/speaking_actions/<avatar_id>/<action_id>/
```

主页面可以刷新说话动作素材，并选择当前说话动作。数字人说话时，会使用选中的动作片段作为身体动作，同时仍然由 Wav2Lip 生成嘴型。

### 2. 静息动作素材库

现在也支持“不说话时播放的静息动作”。素材放在：

```text
data/idle_actions/<avatar_id>/<action_id>/
```

如果没有选择静息动作，数字人会固定在 avatar 的第一帧，避免不说话的时候还一直做讲解动作。如果选择了静息动作，例如 `idle_stand`，数字人不说话的时候会循环播放这段轻微动作。

### 3. 视频处理页面

新增了动作素材制作页面：

```text
http://localhost:8070/motion.html
```

这个页面目前可以做这些事：

- 加载本地视频；
- 预览视频；
- 在时间轴上标记开始点和结束点；
- 把标记好的片段加入队列；
- 给每个片段填写 `action_id`、名称、标签和适合场景；
- 生成说话动作或者静息动作素材；
- 查看素材库；
- 编辑素材信息；
- 删除不需要的素材。

如果要做静息动作，可以打开：

```text
http://localhost:8070/motion.html?kind=idle
```

### 4. 素材删除功能

素材库卡片上已经加了“删除”按钮。删除前会弹确认框，确认后才会删除。

后端新增了：

```text
POST /motion/clips/delete
```

删除时会把对应动作片段的整个文件夹删掉，包括 `full_imgs`、`face_imgs`、`coords.pkl`、`metadata.json` 等文件。如果删的是当前正在使用的动作，后端会把当前选择清掉。

### 5. 动作编排接口

现在已经开始接入“根据讲课文本选择动作”的能力。

新增接口：

```text
POST /motion/plan
```

输入一段讲课文本后，系统会读取当前说话动作素材库，把讲课文本切成几小段，并给每段选择一个 `action_id`。返回结果大概是这样的：

```json
[
  {
    "text": "我们先来看这个概念的定义。",
    "action_id": "explain",
    "display_name": "介绍，讲解",
    "reason": "按关键词规则选择"
  }
]
```

这个接口设计成优先走大模型。如果没有配置大模型 key，就会使用规则兜底。

大模型配置项已经写进 `.env.example`：

```text
MOTION_LLM_BASE_URL=https://api.ikuncode.cc/v1
MOTION_LLM_MODEL=gpt-5.4
# MOTION_LLM_API_KEY=
# MOTION_LLM_TIMEOUT=30
```

真实 key 放到本地 `.env` 后，重启 `8050` 后端，就会切换成大模型编排。

### 6. 主页面动作编排区

主测试页现在新增了“动作编排”区域。流程是：

1. 在 `Text` 文本框里输入讲课内容；
2. 点击“编排动作”；
3. 页面生成动作计划；
4. 点击“按计划讲课”；
5. 系统会按顺序执行：先切换动作，再朗读这一段文本。

目前“按计划讲课”使用的是估算等待时间：一段话提交后，根据文本长度等一会儿，再进入下一段。这个方式可以先跑通流程，但还不是最终最精确的方案。

## 当前限制

现在还有几个限制需要记下来：

第一，大模型目前已经能调用，但 key 只在本地 `.env` 中保存，不会提交到 Git。换机器运行的时候，需要重新配置 `MOTION_LLM_API_KEY`、`MOTION_LLM_BASE_URL` 和 `MOTION_LLM_MODEL`。

第二，动作素材还不够细。现在素材库里主要是讲解类动作，缺少专门的重点强调、提问、总结、左右指向等动作。所以动作编排即使接入大模型，也只能在已有素材里选，不能凭空变出新动作。

第三，按计划讲课目前是按文本长度估算等待时间，不是根据“这一段真正说完了”来切下一段。后面最好改成轮询 `is_speaking`，或者在 TTS / 音频队列里加一个更准确的完成信号。

第四，目前动作编排只看输入文本，不看 PPT 页面，也不看 PPT 中元素的位置。这是当前阶段的有意限制，后面再接 PPT。
